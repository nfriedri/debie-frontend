DEBIE

With the help of this web application it is possible to evalute bias in word embeddings and to remove the bias from them (debiasing).

In the following you will get a short introduction to the topics "word embeddings", "word2Vec algorithms", "implicit and explixit bias", "bias evaluation" and "debiasing"

Embedding spaces

Usage

In the web application DEBIE you can choose between 4 different embedding spaces or simply upload your own embedding space. Fastened to the application are 4 databases containing pre trained word vector representations retrieved through using the fastText, Skip-Gram, GloVe and CBOW models on wikipedia word corpora. The contained vectors all have a dimensionality of 300. For further details on the used vector training models, please follow the links to the references provided in the footer of this web site. By uploading your own embedding space, please pay attention that all vectors have the same dimensionality, otherwise a correct computation can not be ensured. If you want to start multiple requests with your own embedding space please do not upload your embedding space multiple times, instead just copy the name of the database and insert it into the field on top, called "Selected File". The system will then retrieve your file from its memory automatically.

Bias Evaluation

Usage

This webapplication gives the possibility to either evaluate the bias on pre-defined test sets or to evaluate self-defined test sets.

Pre-defined test sets

By retrieving bias evaluations from the pre-defined test sets, simply choose the wanted evaluation methods and the embedding space in the selection bar on top of the site and click on the 'Evaluate'-button. You can choose between the embedding coherence test (ECT), the bias analogy test (BAT), the word embedding association test (WEAT) for explicit bias measuring and the k Means++ clustering algorithm for measuring implicit bias. Alternatively you can simply let the application compute all named measurements, but please be aware that this can take a certain amount of time, depending on the size of the test set. Own defined test and attribute sets

Alternatively you can evaluate your own target and attribute sets under "Upload your own test". Here simply enter your words for the target set seperated by whitespaces into the corresponding fields. Field T1 is for the first target set, field T2 for the second target set, field A1 for the first attribute set and field A2 is for the second attribute set. Choose the wanted evaluations and embedding spaces in the selection bar on top and you are ready to go for evaluating your self defined test.

Debiasing

Usage

This webapplication gives the possibility to either evaluate the bias on the predefined test sets or to evaluate own defined test and attribute sets.

Predefined test and attribute sets

By retrieving bias evaluations from the predefined sets simply choose the wanted evaluation methods and the embedding space you want in the selection bar on top of the site and click on 'Evaluate'. You can choose between the embedding coherence test (ect), the bias analogy test (bat), the word embedding association test (weat) for explicit bias measuring and the kmeans clustering algorithm for measuring the implicit bias. Alternatively you can simply let the app calculate all named measurements, but please be aware that this can take a certain amount of time.

Own defined test and attribute sets

Alternatively you can evaluate your own target and attribute sets under "Upload your own test". Here simply enter your words for the target set seperated by whitespaces into the corresponding fields. Field T1 is for the first target set, field T2 for the second target set, field A1 for the first attribute set and field A2 is for the second attribute set. Choose the wanted evaluations and embedding spaces in the selection bar on top and you are ready to go for evaluating your self defined test.
Additionally you can turn self-defined augmentations on by pressing the augmentations switch-button. Then you have to enter also augmentations on which the debiasing models can be trained.

RESTful API

Usage

For evaluating bias and debiasing it is not necessarily to use this web application as a front-end. In some cases it might be useful to connect directly to the back-end of this application. Therefore, the REST-ful API can be used. The API is accessible via the link: wifo5-29.informatik.uni-mannheim.de:8000/REST/.
